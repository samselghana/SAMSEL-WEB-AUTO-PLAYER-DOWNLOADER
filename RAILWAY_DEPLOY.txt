================================================================================
Deploy SAMSEL Web on Railway
================================================================================

Note: Generic Railway tutorials use main.py and main:app. This project uses
server.py and server:app — Procfile and python -m / __main__ match that.

1. Push this repo to GitHub (or connect the folder).

2. Railway -> New Project -> Deploy from GitHub -> select the repo.
   Root directory: samsel_web (if the repo root is the parent folder, set
   "Root Directory" in Railway service settings to samsel_web).

   If healthchecks fail with "service unavailable": open the **Deploy** log (not
   only Build). Common fixes:
   - Service -> Settings -> **clear Custom Start Command** (must use Dockerfile ENTRYPOINT).
   - Ensure **Root Directory** is the folder that contains server.py + Dockerfile.
   - Docker image uses **requirements-docker.txt** (slim, no librosa) to avoid OOM on small plans.
     For full BPM/genre analysis in the container, change Dockerfile to pip install -r requirements.txt instead.

3. Railway detects Dockerfile + railway.toml and builds the image.

4. Variables (Service -> Variables) — set as needed:

   Required for public Downloader / AutoMix over the internet:
     SAMSEL_AUTOMIX_ALLOW_REMOTE=1
     SAMSEL_AUTOMIX_TOKEN=<long random secret>
   (Do NOT set SAMSEL_AUTOMIX_NO_TOKEN=1 on a public URL.)

   Optional:
     SAMSEL_CORS_ORIGINS=https://your-pages-site.com
     SAMSEL_JINGLE_UPLOADS=0
     SAMSEL_JINGLE_PATH=/app/assets/your-jingle.mp3   (must exist in image or volume)
     SAMSEL_WEB_BUILD=4

5. After deploy, open the Railway URL (e.g. https://your-app.up.railway.app).
   Deploy probe: .../health  (plain "ok" — use this for Railway healthcheck)
   Full JSON:    .../api/health

6. Jingle file: Railway’s filesystem is ephemeral. To ship a default jingle,
   add the MP3 to the repo (e.g. assets/jingle.mp3) and set SAMSEL_JINGLE_PATH
   to that path inside the container, or use a volume (Railway volume mount).

7. Output folder: config still uses server paths; remote sessions use temp dirs
   under the container (see automix_routes). For persistent downloads on the
   server, attach a Railway volume and point output_dir in the UI/config.
